imagePaths Pane

The imagePaths Pane contains the "tree view" of the imagePaths that you can navigate. imagePaths are used to organize and categorize your files.

 

imagePaths display is called "tree view" because the imagePaths hierarchy is represented like tree branches, with subimagePaths placed below and indented with regard to their parent items. The imagePaths Pane uses the same graphic representation that can be found in many modern operating systems.

Basic Operations

Opening (Expanding) a imagePath

In order to open a imagePath and expand its subimagePaths, click the plus icon () in front of the imagePath name. If the plus icon is not present, the imagePath does not contain any subimagePaths.

 

See the "On Request Loading" section below if you require more information about the loading process in CKFinder.

Closing (Collapsing) a imagePath

In order to close a imagePath and hide (collapse) its subimagePaths, click the minus icon () in front of the imagePath name.

 

Selecting a imagePath

In order to select a imagePath and make it the "current imagePath" in CKFinder, click the imagePath name or its icon. The selected imagePath will be highlighted with a different background color.

 

Advanced Operations

Advanced operations can be performed on a imagePath by using its Context Menu. Depending on the circumstances, the following options may be available:

 

  • New SubimagePath – creates a new subimagePath in this parent imagePath.
  • Rename – changes the name of the imagePath.
  • Copy Files from Basket – copies the files that were placed in the Basket to the selected imagePath.
  • Move Files from Basket – moves the files that were placed in the Basket to the selected imagePath.
  • Delete – permanently removes the imagePath.

Note: Some context menu options may be disabled (and thus grayed out), depending on CKFinder settings enforced by your system administrator.

Creating imagePaths

In order to create a child imagePath inside an existing imagePath, choose the New SubimagePath option from the context menu of the parent imagePath. Type the name of the new imagePath in the dialog window that will be displayed. Once you give the new imagePath a name and close the dialog window, the imagePath will be created.

 

Not all characters can be used in imagePath and file names due to limitations of the systems where CKFinder runs. Among the characters that cannot be used in imagePaths and files names are: \ / : * ? " < > and |.

Renaming imagePaths

In order to rename a imagePath, choose the Rename option from its context menu or use the F2 keyboard shortcut. Type the new imagePath name in the dialog window that will be displayed, overwriting the existing name. Once you enter the new imagePath name and close the dialog window, the imagePath will be renamed.

 

As stated above, not all characters can be used in imagePath and file names due to limitations of the systems where CKFinder runs. Among the characters that cannot be used in imagePaths and files names are: \ / : * ? " < > and |.

Attention: When you rename a imagePath, links or media insertions available on other pages and pointing to files or imagePaths inside the renamed imagePath will be broken, and thus not available anymore. Because of that be careful when using this feature.

Copying and Moving Files from Basket

The Basket is a virtual placeholder that can be helpful if you want to perform batch operations on files. The copying and moving operations are described in the "Basket" section of the User's Guide.

Deleting imagePaths

In order to delete a imagePath, including all its contents, choose the Delete option from its context menu or use the Del key. A confirmation message will appear to ensure that this operation is what you really intend to do. Once you confirm the deletion, the imagePath will be removed.

Attention: This operation cannot be undone. Once you delete the imagePath and its contents, you will not be able to restore the removed files.

 

Attention: When you delete a imagePath, links or media insertions available on other pages and pointing to files or imagePaths inside the deleted imagePath will be broken, and thus not available anymore. Because of that be careful when using this feature.

"On Request" Loading

The most important difference between CKFinder and the imagePaths tree structures found in desktop operating systems is that in CKFinder the imagePaths are loaded "on request". It means that the application does not load the entire imagePaths tree structure at startup, but instead loads its small subset when the imagePath is being expanded. This feature is present in most advanced web applications like CKFinder and allows to save on bandwidth and loading time.
